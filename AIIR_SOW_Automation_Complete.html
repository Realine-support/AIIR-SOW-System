Step-by-Step Logic — All Corrections Applied
Step 01
🚀 Trigger
New transcript file uploaded to a designated Google Drive folder triggers the automation. Supports Fireflies auto-upload, Zoom recordings, or manual uploads. File must be readable text (txt, docx, pdf).
Step 02
📄 Ingest & Validate
Parse transcript to raw text. Validate it is non-empty and readable. If invalid, alert team and stop. No further steps run on a bad file.
Step 03
🔍 Extract Core Variables Updated
NLP extraction of all key variables:
  • Client name (used in 3 SOW locations)
  • HubSpot Deal ID → header AND footer
  • Coach name (only post-chemistry call)
  • Leadership seniority level
  • Market type: Mature or Emerging — affects bill rate
  • Duration signals (months)
  • Number of participants → E49
  • Budget signals & price anchors
  • Existing 360 / assessment status
  • 'Client' vs 'coachee' terminology
Step 04
🎯 Select Coaching Tier Updated
Each tier has structurally different sessions — not just different hours. SPARK I has no Dev History interview at all. ROADMAP includes LD12 360° assessment. ASCENT & VISTA use $50 CZ rate. See Tiers tab for full breakdown.
  • ASCENT — C-Suite, 10–12 months
  • IGNITE — Senior, 6–9 months
  • ROADMAP — Mid-level, 4–6 months
  • SPARK I — Early career, 3–4 months
  • SPARK II — Early career, 4–5 months
  • AIIR VISTA — Executive advisory, 6 or 12 mo
Step 05
💵 Set Bill Rate — B15 New Step
Bill rate is a manual input — not automatic. Must detect both market type AND seniority from transcript, then write to B15:
  • Mature + Tier 1/Senior → $350/hr
  • Mature + Tier 2 → $250/hr
  • Mature + C-Suite → $600/hr
  • Emerging + Tier 1 → $300/hr
  • Emerging + Tier 2 → $200/hr
  • Emerging + C-Suite → $500/hr
Step 06
📊 Set Margin — B16 Corrected
Default margin is 65% (0.65), not 70%. The Excel file currently has 70% (0.7) but Andrew's comment on cell B16 explicitly flags Megan uses 65%. Write 0.65 to B16 as the default unless a different margin is agreed with the client.
Step 07
💰 Budget Signal Detection
Keyword scan of transcript for cost constraint signals. If detected, extract the target budget amount. If no signals found, use all default hours for the selected tier. See Keywords tab for full list.
Step 08
⚙️ Apply Reduction Hierarchy Updated
Apply reductions in Megan's priority order. 6 levers total — not 5. Development Planning Session was previously missing as a separate lever. See Hierarchy tab.
  • 1. Stakeholder sessions → first
  • 2. Dev History Interview → second
  • 3. 360° Interviews → third (with sub-tree)
  • 4. Implementation Sessions → fourth
  • 5. Dev Planning Session → last resort A (can remove entirely)
  • 6. Assessment Feedback → last resort B
Step 08b
🔄 360° Sub-Logic Updated
Minimum is 5 interviews — hard floor (not "5-6"). Two reduction types: reduce time per interview (45→30 min) OR reduce count (8→5 min). Eliminate entirely only if existing 360 data available. If eliminated: set B40=0 AND update SOW "three streams" → "two streams."
Step 09
📊 Write to Excel Updated
Write to tier-specific cells. E49 (# participants) is blank — must be populated. SOW price comes from H50 (H48 × E49). If single participant, H48 = H50. See Excel Cell Map tab for every cell per tier.
Step 10
💳 Payment Terms Detection Corrected
Default is 100% upfront at kickoff — NOT 50/50. The 50/50 split is only applied when the client explicitly negotiates it. 45-day net terms require CFO approval. 30 days is the default net term.
Step 11
📋 Generate Pricing Draft
Produce human-readable pricing summary with annotated rationale for every change made — which keyword triggered it, what changed, and what the new price is. Sent to team for review before SOW generation.
Step 12
👤 Human Review — Pricing
Team member reviews AI-generated pricing in Excel/Google Sheet. Can manually adjust any cell. Must explicitly approve (button click, status cell, or file move) before SOW generation is triggered.
Step 13
📝 Populate SOW Template Updated
Inject variables. Client name appears in 3 separate locations (body paragraph, policy section ×2, signature block). See SOW Field Map tab for exact rules per field. Key rules:
  • Boilerplate intro → NEVER change
  • Policies → NEVER change
  • If 360 eliminated → "two streams" not three
  • Dev History duration → update or remove mention
  • Assessment Feedback → update if 90 min OR delete sentence if 2hr
Step 14
👤 Human Review — SOW
Team reviews full SOW draft in Google Docs or Word. Edits directly if needed. Final approval triggers send to client.
Step 15
📧 Send & Archive
Approved SOW sent to client via email or DocuSign. Archived to Client Master Folder in Google Drive with standardized naming. HubSpot deal status updated.
